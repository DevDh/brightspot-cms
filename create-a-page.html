Once objects are modeled, and published in the CMS, the next step is to build pages on which to display them.

Start by adding rendering logic to an already created object. Add two annotations to the object, one to define the JSP that renders the object content, and another that creates the page structure to be associated with it. Note, users of Brightspot 2.0 should see the template guide for that version, which is also available.

Add Renderer Paths

{% highlight java %}@Renderer.LayoutPath("/layouts/article-template.jsp") @Renderer.Path("/article-object.jsp") public class Article extends Content { private String headline; private ReferentialText body; private Author author; // Getters and Setters } {% endhighlight %}

Create JSP Files

To start, create two files to render the page, the first is the entire page layout. As per the path specified within the annotation on the example above, create an article-template.jsp file in a new directory within webapp, called "layouts". Start by importing taglibs and then create a simple structure for your page. You can add a header, to render the headline, and a content section, which will render the rest of the object.

Adding Sections

Page Templates are typically made up of multiple sections. Brightspot allows the creation of individual sections, to be used on one, or shared on many page templates.

Every page template contains a defined Main Content type. In the example above, the basic page displays an Article object. To illustrate the use of another content type on the same page template, create a new object, and render it on the same page as content. The new object in the example below is a list of top authors. The object simply contains a list of Authors, editorially created within the CMS. Create the new object in your text editor or IDE, and refresh or rebuild your project. Then publish a version in the CMS. Click into Search > Create New and find the Top Authors in the drop-down.
