1. Work Streams

Work Streams allow editors to group together content, and then work through the editing of it quickly. A new work stream is created directly from a set of search results.

2. Advanced Filters

Dashboard widgets now have improved filtering, with content shown based on a specific user, or role. Also, global filters, such as tags or topics can be designated and used to filter content at a high level.

3. Resource Widget

New to the dashboard is a Resources widget, which provides a way for editors to add useful files or links that are always accessible from within the CMS. Links or files are added in Admin > Settings.

4. Updated User Profiles

Each user now has a profile with preferences for time-zone, notifications and the ability to set two factor authentication.

5. Customize Create New

The Create New widget can now be customized, per user. Hide templates you don't create regularly to tidy your dashboard view.

6. Bulk Upload

The bulk uploader now has a dedicated widget on the dashboard, where you can select, or drag and drop files you want to upload in bulk.

Brightspot 2.1 Developer Release Notes

Metrics

Brightspot now includes a powerful new datatype: Metric, which provides the ability to programmatically record statistically interesting data such as page views, votes, etc. The values can then be queried with the Dari Query API. Aggregation of metric data over time can achieved, answering questions like "sort my posts by number of likes in the past 24 hours, week or all time."

Support for W3 Grid Layout Specification

Brightspot now implements a subset of the W3 Grid Layout specification for defining grid layouts. This provides a flexible way to define grid layouts using a compact CSS format. Find out how to use the new grid layout in the dedicated documentation section.

New Annotations

Several new annotations have been added, including a new Tab user interface. See all the new annotations in the documentation.

Custom Widgets

With 2.1 Developers can now add custom Dashboard widgets, to appear on the user dashboard.

1. Inline Guides

Fields with specific rules, such as limits or requirements now have a `?` pop-up, that provides more details, helping editors as they publish.

2. Text Overlay Tool

A new text overlay tool has been added, allowing rich text to be added to any image and moved to where the editor desires.Multiple text areas can be added to each image, and each crop can contain a new set of text overlays. Custom CSS can be added to the CMS to control the text styles.

3. Create New & Copy

When creating multiple pieces of content that share similar properties, with slight variations in a field, an exact copy can be made. The tool can be useful in speeding up editorial tasks. An editor can create a new instance of the object with all the previous object fields intact. These can then be edited as required, and published as a new instance.

4. Advanced Image Editing

Editors now have the ability to apply numerous advanced edits to an image within Brightspot. Brightness, contrast and filters can be chosen, as well as orientation changed.

5. Image Crop Update

The user interface for cropping images has been updated, with a preview of each crop now shown. Dragging and changing the crop area updates the preview automatically. Crops with the same aspect ratio can now be grouped together, to make the interface cleaner.

6. Word & Character Counter

All text fields and rich text areas now have a word and character counter within them, helping editors keep track of their proximity to limits, if any apply.

1. Persisted Search

Searches are now persisted within Brightspot, allowing editors to quickly access results they just came from, right within the content edit screen. Working through a set of results just got easier thanks to this added feature.

2. New Rich Text Editor

The original Rich Text Editor for Brightspot, CKEditor, has now been replaced with WYSIhtml5. The intuitive user interface remains unchanged, and formatting irregularities seen with CKEditor are resolved. The link widget within the rich text has been replaced, and issues surrounding adding enhancements have been fixed. Also, when pasting from Microsoft Word formatting bugs have been resolved.

3. Preview Everywhere

All pages can now be previewed inline within the CMS. Click on the eye icon beside a page link to see a live preview. This can be used to preview scheduled, drafts, or even new content.

Editors can also take advantage of an Inline Preview tool, showing their page content live, within the content edit screen. Changes are reflected immediately in the preview view. A 'Live Page' link, to access the url in browser as well as the standard 'Share' tool are also included in the new layout. The preview can be hidden for editors wishing for the classic preview pop-up view.

Inline Editing

Often editors want to review their web content visually, not from within the CMS interface. When doing so, Brightspot now gives you the ability to edit the content from a web view, without having to navigate back into the CMS to make changes.

The inline editor allows live content to be updated, images changed and if needed, full CMS access at a single click. The tool is accessed via a link within your browser. Start by logging into the CMS. Once logged in, open your site and navigate to a page to start using.

Developer Release Notes

Brightspot CMS version 2.0 contains many improvements focused on making life even easier for the developer. If you want to know more about any of the items in this section see the documentation, JavaDocs or ask us a question.

Wireframe Tool

Available within the inline preview on a page the new wireframe tool uses the page template to create a wire framed visual of any page. Section names and layouts are used to produce a wireframe mock of your page layout. This feature is available from any Brightspot built page, by adding ?_wireframe=true to a URL. Note, Production mode must be off ?_prod=false. Within each page section the content chosen can be accessed directly. The tool helps to highlight any gaps in development, as well as providing a detailed list of all the JSTL hooks available for each section. The drop down menus for each section highlight the properties available on that page section. You can see the tool in the dedicated CMS Tools section

Fields Clause

Specific fields can now be returned directly from a Dari object, without the need to return the entire object. See the Querying section for more.

Improved Debug Tools

The entire _debug tool has had a refresh. The code tool now includes a sandbox, and within the UI object types can be found through typeahead search.

Stats Tool

Recently added to the Dari Debug tools is a Stats Tool. It provides insight into the performance of Dari when running inside a servlet container. It provides information on SQL and Solr Throughput and Latency as well as JSP includes, averages and HTTP Response Throughput and Latency. See more in the dedicated Dari Tools section.

Custom Field Index Tables

Custom Field Index Tables now allow developers to access data in database tables other than the standard Dari Record tables. Read the Field Index Tables documentation to see how to start using the feature.
