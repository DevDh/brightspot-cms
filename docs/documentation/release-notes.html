Brightspot 2.2 Developer Release Notes

Contextual Rendering

Often an object will be used in multiple places throughout a website, with different rules around how it is displayed for each location. In Brightspot 2.2 you can now provide multiple context rules, so that an object can be rendered differently, based on the context. See the Contextual Rendering documentation to start using this feature.

Directory.Item

Templates and Pages have been deprecated in Brightspot 2.2. With the use of the LayoutPath and Path rendering annotations, objects themselves can be defined as a page or template by simply implementing Directory.Item. A full overview can be seen in the Create a Page documentation section.

Syndication

Content can now be easily shared across Brightpot powered sites. Modules can be syndicated to appear on other sites using the new Embed annotation. See the Syndication documentation for an overview.

Page Stage

While a global page container may work for most pages on your site, sometimes you want to insert custom styles for a specific page. To enable this a new feature, PageStage has been added to Brightspot 2.2. This class is commonly used to easily update and render the contents of the element. Learn how to start using this feature by reading the Page Staging documentation.

Color Image

Editors now have the ability to search for images by color. To enable this feature implement ColorImage in your Image class. See the Image documentation for updating to use this feature.

New Annotations

There are several new annotations now available in Brightspot 2.2. See the annotations section for a full list. Notable additions include the ability to add new tabs for content using @ToolUi.Tab("tabName") and the @ToolUi.DisplayFirst annotation, which allows fields added via a modification to be ordered in the content type. The @ToolUi.CodeType annotation has also been added, which allows text fields to be turned into code input fields.

Brightspot 2.1 Developer Release Notes

Metrics

Brightspot now includes a powerful new datatype: Metric, which provides the ability to programmatically record statistically interesting data such as page views, votes, etc. The values can then be queried with the Dari Query API. Aggregation of metric data over time can achieved, answering questions like "sort my posts by number of likes in the past 24 hours, week or all time."

Support for W3 Grid Layout Specification

Brightspot now implements a subset of the W3 Grid Layout specification for defining grid layouts. This provides a flexible way to define grid layouts using a compact CSS format. Find out how to use the new grid layout in the dedicated documentation section.

New Annotations

Several new annotations have been added, including a new Tab user interface. See all the new annotations in the documentation.

Custom Widgets

With 2.1 Developers can now add custom Dashboard widgets, to appear on the user dashboard.

Developer Release Notes

Brightspot CMS version 2.0 contains many improvements focused on making life even easier for the developer. If you want to know more about any of the items in this section see the documentation, JavaDocs or ask us a question.

Wireframe Tool

Available within the inline preview on a page the new wireframe tool uses the page template to create a wire framed visual of any page. Section names and layouts are used to produce a wireframe mock of your page layout. This feature is available from any Brightspot built page, by adding ?_wireframe=true to a URL. Note, Production mode must be off ?_prod=false. Within each page section the content chosen can be accessed directly. The tool helps to highlight any gaps in development, as well as providing a detailed list of all the JSTL hooks available for each section. The drop down menus for each section highlight the properties available on that page section. You can see the tool in the dedicated CMS Tools section

Fields Clause

Specific fields can now be returned directly from a Dari object, without the need to return the entire object. See the Querying section for more.

Improved Debug Tools

The entire _debug tool has had a refresh. The code tool now includes a sandbox, and within the UI object types can be found through typeahead search.

Stats Tool

Recently added to the Dari Debug tools is a Stats Tool. It provides insight into the performance of Dari when running inside a servlet container. It provides information on SQL and Solr Throughput and Latency as well as JSP includes, averages and HTTP Response Throughput and Latency. See more in the dedicated Dari Tools section.

Custom Field Index Tables

Custom Field Index Tables now allow developers to access data in database tables other than the standard Dari Record tables. Read the Field Index Tables documentation to see how to start using the feature.
