Creating New Content

When using Brightspot, creating a new object automatically generates a user interface within the CMS, where editors can interact and publish content. In this section we are going to look at how to model your objects, and the corresponding UI that is created within the CMS.

Create an object

Having installed Brightspot, you can start to create objects to appear within the CMS. To do so, create a Java class, and extend Content. Save your object within your project at src/main/java/yourGroupID. Once saved, refresh your CMS browser window and click into the Search tool. In the Create drop-down you should see your new object. Click New to open it within the content edit view. The name of your Java class will appear in the CMS as the name of the object. Below is an example Article object. First take a look at the Java class, then below, the corresponding UI created within the CMS:

{% highlight java %}package com.psddev.brightspot; import com.psddev.cms.db.*; import com.psddev.cms.tool.*; import com.psddev.dari.db.*; import com.psddev.dari.util.*; import java.util.*; public class Article extends Content { private String headline; private Author author; private ReferentialText body; private List tags; private Set keywords; }{% endhighlight %}

Adding Fields

There are a range of standard UI elements that each object can use. Start by adding some to your newly created object. Refresh the CMS to see the new UI appear. Remove the code within your Java class to remove the UI element. You can reorder the UI by changing the order of the fields in the Java class. The name of your field will be reflected directly in the CMS as the UI label. Below we have called out some the UI elements within our Article object. See the full list available here.

Associating Objects

Once you have your first object created you can add a second. In our example we have an Author object that is associated with each Article. Associate an object by naming it as a return type in your Java class.

{% highlight java %}public class Author extends Content { @Required private String name; private StorageItem file; @Embedded private Address address; private Boolean bestSeller; private Date dateOfBirth; } {% endhighlight %}
Annotations
Add a @Required annotation to make the field required.
Embedded Objects
Other objects can be embedded using the@Embedded annotation
Calendar Widget
Use theDate return type to add a calendar tool.
Boolean
StandardBoolean or enum can be added
File Upload
Use StorageItem return type to create a file uploader


Dari Schema Tool

With several objects created you can leverage the Dari Schema Tool, which gives an instant view of your data model. Navigate to http://localhost:8080/_debug/db-schema and select one of your objects.
